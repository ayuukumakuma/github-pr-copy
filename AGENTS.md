# Repository Guidelines

## 目的
このリポジトリは小規模な Chrome 拡張です。ガイドでは、コードを読めば分かる実装詳細ではなく、変更時の判断基準と運用上の注意だけを扱います。

## 構成方針
現状は少数ファイルで完結しているため、無理に階層化しません。機能追加で責務が増えた場合のみ、DOM 操作、GitHub ページ判定、コピー処理のように関心ごとごとへ分割してください。

アセットは `images/`、拡張設定は `manifest.json`、主要ロジックは JavaScript ファイルに置きます。新規ファイルを足す場合は、用途が名前だけで分かる粒度にしてください。

## 開発と確認
ビルド工程はありません。`chrome://extensions/` で `Load unpacked` を使って読み込み、変更後は拡張を再読み込みして確認します。

確認は GitHub の PR ページで行い、少なくとも以下を手動で見ます。

- ボタンが重複せずに表示されること
- コピー結果が期待どおりであること
- GitHub 側の部分更新後も表示が崩れないこと

コミット前には `git status --short` で差分を確認してください。

## コーディング規約
JavaScript は 2 スペースインデント、セミコロンありを維持します。共有定数は意味が即座に分かる名前にし、処理は小さな関数へ分けてネストを浅く保ってください。

このリポジトリでは、抽象化のしすぎより追いやすさを優先します。ブラウザ上で不具合を追えることを重視し、暗黙的な制御や過度な共通化は避けます。

## テスト方針
自動テストは未整備です。変更ごとに手動確認を必須とし、表示タイミング、再描画耐性、クリップボード関連の挙動を優先して確認してください。

将来的にテストを追加する場合は `tests/` を作成し、対象に対応する名前を付けます。例: `contentScript.test.js`

## コミットと PR
コミットメッセージは Conventional Commits ベースで統一します。例: `fix: PRコピーボタンの表示安定性を改善`

PR には変更概要、手動確認手順、UI 変更時のスクリーンショットまたは短い録画を含めてください。権限追加や対象 URL 変更のような利用範囲に関わる変更は、理由を PR 本文に明記します。
